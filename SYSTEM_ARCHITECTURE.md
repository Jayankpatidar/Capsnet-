## Security Features

```
ğŸ”’ SECURITY LAYERS:

   - bcrypt hashing (rounds: 10)
   - Never stored plain text
   - Compared using bcrypt.compare()

   - Tokens expire after 7 days
   - Verification tokens one-time use
   - Token type validation

   - Email must be verified before login
   - Verification link has expiration

   - HTTP-only cookies (prevent XSS)
   - Secure flag in production (HTTPS)
   - SameSite: lax (prevent CSRF)

   - All user routes protected by auth middleware
   - Token validation on every request
   - User ID extracted from token, not from request
```
